# Template Helpers

Jinja2 helpers available in component assembly templates (`.s` and `.inc`
files under `breadbox/`). Each helper generates a mangled ca65 symbol name and
communicates the **intent** of the usage site.

## Quick Reference

| Helper         | Returns              | Registers | Used at                              |
|----------------|----------------------|-----------|--------------------------------------|
| `api_def(n)`   | `__{P}_{n}`          | Yes       | `.proc` — public API definition      |
| `api(n)`       | `__{P}_{n}`          | No        | `jsr` — public API reference         |
| `my_def(n)`    | `__{P}_{n}`          | No        | `.proc` — internal proc definition   |
| `my(n)`        | `__{P}_{n}`          | No        | `jsr`, `.constructor`, `.interruptor`|
| `zp_def(n)`    | `__{P}_{n}`          | Yes       | label `.res` — exported ZP variable  |
| `zp(n)`        | `__{P}_{n}`          | No        | `lda`/`sta` — exported ZP reference  |
| `var(n)`       | `__{P}_{n}`          | No        | label/`lda`/`sta` — internal data    |

`P` is the component's `scope` (e.g. `UART`, `DISPLAY`, `LED_PIN`).

## What the Generator Auto-Generates

The generator uses metadata collected from `api_def()` and `zp_def()` calls
during template rendering to produce two things automatically:

1. **`.export` / `.exportzp` directives** — injected into the rendered `.s`
   file, before the first `.segment` directive.
2. **`api.inc`** — the public API include file with `.scope`, `.import` /
   `.importzp`, and alias assignments.

Template authors never write `.export`, `.exportzp`, or `api.inc` by hand
(exception: internal exports like gpio_group's `tmp` use manual
`.exportzp {{ var("tmp") }}`).

## Helper Details

### `api_def(name)` / `api(name)` — Public API

Public subroutines exposed to user code via the `.scope` block in `api.inc`.

```jinja2
{# definition — at .proc site: #}
.proc {{ api_def("init") }}
    ...
.endproc

{# reference — calling a public API from within the component: #}
jsr {{ api("write") }}
```

### `my_def(name)` / `my(name)` — Internal Procs

Private subroutines internal to the component. Not exported, not in `api.inc`.

```jinja2
{# definition — at .proc site: #}
.proc {{ my_def("pulse_en") }}
    ...
.endproc

{# reference — internal call: #}
jsr {{ my("pulse_en") }}

{# .constructor / .interruptor use my() because the CONDES mechanism
   is internal to the component, even when the proc itself is public: #}
.constructor {{ my("init") }}, 16
.interruptor {{ my("irq_handler") }}
```

### `zp_def(name)` / `zp(name)` — User-Facing Zero-Page

Exported zero-page variables that form part of the component's public API
(e.g. a pointer the user sets before calling `print`).

```jinja2
{# definition — at .res label site: #}
.segment "ZEROPAGE"
{{ zp_def("ptr") }}:  .res 2

{# reference: #}
lda ({{ zp("ptr") }}),y
```

### `var(name)` — Internal Data

RAM buffers, internal ZP temporaries, status caches — any data symbol that
is not a subroutine and not part of the public API.

```jinja2
{# definition: #}
.segment "KERNALRAM"
{{ var("rx_buffer") }}: .res $100

.segment "ZEROPAGE"
{{ var("tmp") }}: .res 1

{# reference: #}
lda {{ var("rx_buffer") }},x
sta {{ var("tmp") }}

{# internal export for macro use (manual, not auto-generated): #}
.exportzp {{ var("tmp") }}
```

### Plain Constants

Hardware registers and configuration values use plain assembly equate names
with Jinja2 only for computed values. No helper function needed.

```jinja2
{# definition: #}
CTRL_BAUD = {{ baud_code | hex }}
STATUS    = {{ (address + 1) | hex }}

{# reference: #}
lda #CTRL_BAUD
sta STATUS
```

## Addressing Modes

Index registers go **outside** the helper call:

```jinja2
lda {{ var("rx_buffer") }},x      {# correct #}
lda {{ var("rx_buffer,x") }}      {# wrong #}
```

## ld65 CONDES Symbols

`__CONSTRUCTOR_TABLE__`, `__CONSTRUCTOR_COUNT__`, `__INTERRUPTOR_TABLE__`,
`__INTERRUPTOR_COUNT__` are generated by ld65 and stay hardcoded.

## CORE Component Exception

The CORE component does not use `api_def()` — its `api.inc` is a
hand-maintained template that includes sub-files (`boot.inc`, `delay_macros.inc`,
etc.). The generator falls back to rendering this template when no
`api_def()` calls are registered.
